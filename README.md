# System Design социальной сети для курса по System Design
## Требования к системе
### Функциональные требования
Пользователь может:
- Публиковать посты
- Оставлять комментарии к постам
- Оценивать посты и комментарии других пользователей
- Подписываться на уведомления об активности (публикация постов) других пользователей
- Посмотреть ленты других пользоватsелей (DESC)
- Посмотреть ленту аггрегированную из постов пользователей на которых активна подписка

### Нефункциональные требования
- DAU = 10000000
- Локация - СНГ
- Availability = 99.99 %
- post size ~ 5 MB

Действия пользователей (для оценки):
* Публикация поста: in, 1 rpd
* Комментарии к постам: in, 5 rpd
* Оценки: in, 10 rpd
* Подписка: in, 1 rpd
* Просмотр ленты (пагинация по 20 постов): out 100 постов (5 запросов) в день

Расчет:
- RPS ≈ DAU \* действия 1 пользователя / 86400 = 1e+7 * (1+5+10+1+5) / 86400 = 2546
- Max RPS ≈ RPS * 5 = 12730
- traffic in ≈ 1e+7 \* 1 \* (5 + 1) / 86400 = 694 MBps = 5552 Mbitps
- traffic out ≈ 1e+7 \* 100 \* 5 / 86400 = 57870 MBps = 462960 Mbitps
